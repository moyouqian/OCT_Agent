# OCT Agent 修改计划与记忆规划

## 项目定位

OCT Agent 是一个通用科研助手，不是应变计算专用工具页。当前主流程保持“自然语言对话 + 附件 + artifact 结果面板”，应变计算只是其中一个能力；后续会继续扩展文献检索、文献总结、实验记录整理和报告生成。

## 本轮修复计划

- 回答空白过多：压缩 AI 回复中的连续空行，并收紧 Markdown 标题、段落、列表和分割线间距。
- 热力图过长：侧栏模式使用较短的热力图高度，避免一张图片把结果卡片拉得过长。
- 结果面板全屏：右侧结果面板增加展开/收起按钮，展开后覆盖整个页面，方便查看热力图。
- 新图替换旧图：新一轮可能生成图片的任务会清空旧结果，后续只展示本轮最新图片。

## 前端修改要点

- AI 回复渲染前将连续 3 个以上换行压缩为 2 个，保留正常 Markdown 段落。
- `.markdown-body` 不再继承消息气泡的 `pre-wrap`，避免模型输出中的多余空白被原样放大。
- `ResultGallery` 维护展开状态，默认侧栏展示，展开后使用全屏 artifact 查看模式。
- `HeatmapViewer` 根据侧栏/全屏模式设置不同 Plotly 高度：侧栏更紧凑，全屏更适合查看细节。
- 前端提交可能生成结果的任务时清空旧 `results`，下一批 `result_refs` 替换旧结果；普通聊天不清空结果。

## Agent 记忆模式规划

- v1 使用本地轻量记忆文件 `data/memory.json`，该文件不进入 Git。
- 记忆分为四类：用户偏好、项目事实、常用物理参数、文件/实验摘要。
- 支持自然语言指令：“记住… / 忘记… / 查看记忆”。
- 每次对话只注入少量相关记忆摘要，不把完整记忆塞进用户消息。
- 文献检索能力上线后，文献摘要既可以作为 artifact，也可以由用户确认后进入长期记忆。

## 验收标准

- `npm.cmd run build` 通过。
- 普通问答如“介绍广州美食”显示紧凑排版，不再出现大块空白。
- 侧栏热力图高度缩短，控件、色条、坐标轴和图像不遮挡。
- 点击结果面板展开按钮后，结果放大为全页面查看；收起后回到聊天 + 侧栏布局。
- 连续运行两次应变计算时，第二次结果出现后右侧只显示第二次图片。
- 普通聊天消息不会清空当前结果图。
- `frontend/src` 和 `AGENT.md` 不残留典型乱码。
